# Chess PCA Morphological Framework (Pipeline + Figures)

This repo contains a modular Python pipeline to:
- extract per-move features from chess PGN games (White/Black perspectives),
- run a unified PCA analysis,
- generate paper/appendix figures (H1–H4) and loading profiles.

## Project structure

- `pipeline.py` – end-to-end orchestration (dataset cache, unified PCA, figure generation)
- `io_pgn.py` – PGN / PGN.GZ streaming ingestion
- `features.py` – feature extraction from chess positions (python-chess)
- `analysis.py` – PCA + trajectories utilities (unified PCA, trajectory aggregation)
- `plotting.py` – shared plotting utilities for unified trajectory panels
- `figure_loadings.py` – loading tables + loading-profile plots (PC1/PC2, White/Black)
- `figure_h1.py` – scree + null models for EVR(PC1+PC2)
- `figure_h2.py` – bootstrap stability of PCA components (cosine similarity)
- `figure_h3.py` – geometric drift on PC1 (Δz1) + stats
- `figure_h4_residence.py` – residence time in outcome-aligned attractor basins

### Data / outputs (recommended)
Use the following convention (you can adapt paths inside `pipeline.py`):
- `data/raw/` – input PGN / PGN.GZ files
- `data/cache/` – cached datasets (parquet) generated by the pipeline (ignored by git)
- `data/outputs/` – generated figures and tables
- `data/outputs/final_figures/` – final PNGs you want to commit and show on GitHub
